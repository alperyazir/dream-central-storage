## Dev Notes

### MUI → shadcn Component Mapping
| MUI Component | shadcn/ui Replacement |
|---|---|
| Dialog, DialogTitle, DialogContent, DialogActions | Dialog, DialogHeader, DialogTitle, DialogContent, DialogFooter |
| TextField | Input + Label |
| Select, MenuItem | Select, SelectTrigger, SelectContent, SelectItem |
| FormControlLabel + Switch | Switch + Label |
| Checkbox | Checkbox |
| LinearProgress | Progress |
| Chip | Badge |
| Tabs, Tab | Tabs, TabsList, TabsTrigger, TabsContent |
| Alert | Alert |
| Slider | Slider |
| Divider | Separator |
| Collapse | Collapsible (from Radix) or conditional render |

### Important: Preserve Business Logic
All API call logic in the `lib/` directory remains unchanged. Only the UI rendering layer changes. Keep:
- `lib/uploads.ts` - upload API calls
- `lib/publishers.ts` - publisher CRUD
- `lib/teachers.ts` - teacher CRUD
- `lib/processing.ts` - processing API calls
- Error handling patterns (try/catch, ApiError parsing)

### Dependencies
- Story 11.1 (foundation), 11.2 (layout), 11.3 (login - Card, Input, Button already created)
- These components are used by pages in stories 11.5-11.9
