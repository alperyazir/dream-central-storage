# Story 1.2: S3 Storage Integration

## Status
Ready for Review

## Story
As a developer, I want to configure the API to securely connect to the MinIO S3-compatible object store, so that the application can access the designated storage bucket for file operations.

## Acceptance Criteria
1. The API retrieves S3 connection credentials (endpoint, key, secret, bucket name) from environment variables.
2. The API includes a health check or startup process that verifies a successful connection to the S3 bucket.
3. Connection errors are properly logged.

## Tasks / Subtasks
- [x] Add centralized configuration for S3 env vars
  - [x] Implement `app/core/config.py` to load `S3_*`/`MINIO_*` env vars
- [x] Implement MinIO client and connectivity check
  - [x] Add `app/services/storage.py` with client creation and `check_s3_connection`
- [x] Wire startup connectivity check
  - [x] Extend `app/main.py` lifespan to verify S3 and log status
- [x] Add storage health endpoint for observability
  - [x] `GET /storage/health` returns JSON with connection status and detail
- [x] Update README with required env vars and endpoint
- [x] Validate locally against a MinIO instance
- [x] Add integration tests with a MinIO test container

## Dev Notes
