# Story 11.1: Foundation Setup - Tailwind CSS & shadcn/ui Installation

## Status
Ready for Review

## Story
**As a** developer,
**I want** the admin panel migrated from MUI/Emotion to Tailwind CSS v4 and shadcn/ui,
**so that** we have a modern, utility-first styling foundation for the dashboard redesign.

## Acceptance Criteria
1. Tailwind CSS v4 installed and configured with Vite plugin
2. shadcn/ui initialized with base configuration (components.json, cn utility)
3. CSS variables theme defined with OKLCH color system (light + dark mode)
4. Existing turquoise (#2cb8ad) and denim (#1b79b0) brand colors mapped to new theme
5. MUI, @mui/icons-material, @emotion/react, @emotion/styled removed from package.json
6. Lucide React installed as icon replacement
7. App still compiles (placeholder wrappers are acceptable for this story)
8. tailwind.css replaces all 4 existing CSS files (global.css, layout.css, navbar.css, page.css)

### cn() Utility
```typescript
import { clsx, type ClassValue } from "clsx"
import { twMerge } from "tailwind-merge"

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```
