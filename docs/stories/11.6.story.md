## Dev Notes

### Common Table Pattern
All table pages follow this layout:
```
<div class="space-y-4">
  <!-- Header with title + action button -->
  <div class="flex items-center justify-between">
    <h1 class="text-2xl font-semibold">Page Title</h1>
    <Button>Add New</Button>
  </div>

  <!-- Filters bar -->
  <div class="flex items-center gap-3">
    <Input placeholder="Search..." class="max-w-sm" />
    <Select>Filter</Select>
  </div>

  <!-- Optional: Tabs -->
  <Tabs>
    <TabsList><TabsTrigger>Active</TabsTrigger><TabsTrigger>Trash</TabsTrigger></TabsList>
    <TabsContent>
      <Card>
        <Table>...</Table>
      </Card>
    </TabsContent>
  </Tabs>
</div>
```

### Sortable Header Pattern
```tsx
<TableHead
  className="cursor-pointer select-none"
  onClick={() => handleSort("column")}
>
  Column Name
  {sortField === "column" && (sortDir === "asc" ? <ChevronUp /> : <ChevronDown />)}
</TableHead>
```

### MUI Chip → shadcn Badge
Status chips become Badge variants:
- `<Chip label="Active" color="success" />` → `<Badge variant="default">Active</Badge>`
- `<Chip label="Inactive" color="warning" />` → `<Badge variant="secondary">Inactive</Badge>`
