# Story 1.1: Initial Project & API Setup

## Status

Done

## Story

As a developer, I want to set up the new Python FastAPI project with the correct structure and a basic health-check endpoint, so that we have a verifiable and deployable foundation for the API service.

## Acceptance Criteria

1. A new FastAPI project is initialized with a standard structure (`app/main.py`, `app/api/`, etc.).
2. `GET /health` returns `200 OK` and JSON exactly `{ "status": "ok" }`.
3. The app runs locally; README documents `uvicorn app.main:app --reload` and a curl example.
4. Code style is enforced: Black and Ruff configured (e.g., in `pyproject.toml`) and pass on the scaffold.
5. A unit test for `/health` exists and passes (pytest).

  - [x] Create service repo `dream-central-storage-api` with `app/` package and `app/main.py` defining `app = FastAPI()`.

  - [x] Add tool config (Black/Ruff). Optional `Makefile` can be added later if needed.
