# Story 11.7: Detail Pages Migration (PublisherDetail, TeacherDetail, BookExplorer)

## Status
Done

## Story
**As an** admin user,
**I want** the Publisher Detail, Teacher Detail, and Book Explorer pages rebuilt with shadcn/ui,
**so that** I can view detailed information about publishers, teachers, and book contents in a modern interface.

## Acceptance Criteria
1. PublisherDetail: header with back button, publisher info Card with logo, books Table (searchable, sortable, with covers), assets section with collapsible file tree, activity breakdown popover
2. TeacherDetail: teacher info Card (ID, name, email, status), storage insights Card (by-type breakdown), sortable materials Table (name, type, size, date, AI status), file type icons
3. BookExplorerDrawer: slide-out drawer/sheet from right side, file tree browser, preview support (image, audio, video), download button, config metadata display
4. Back navigation works on detail pages
5. All existing data fetching and display logic preserved
6. Collapsible sections animate smoothly

```
  </div>

  <!-- Info + Assets grid -->
  <div class="grid grid-cols-1 gap-6 lg:grid-cols-3">
    <Card class="lg:col-span-1">Publisher Info</Card>
    <Card class="lg:col-span-2">Assets Tree</Card>
  </div>

  <!-- Books Table -->
  <Card>
    <CardHeader><Input placeholder="Search books..." /></CardHeader>
    <CardContent><Table>Books</Table></CardContent>
  </Card>
</div>
```

### File Tree Pattern
```tsx
<Collapsible>
  <CollapsibleTrigger>
    <ChevronRight className="transition-transform data-[state=open]:rotate-90" />
    <Folder /> folder-name
  </CollapsibleTrigger>
  <CollapsibleContent className="ml-4">
    {children}
  </CollapsibleContent>
</Collapsible>
```
