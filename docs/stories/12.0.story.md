# Story 12.0: Codebase Cleanup

## Status
Ready for Review

## Story
**As a** developer preparing for production deployment,
**I want** the codebase cleaned of debug artifacts, dead code, and formatting inconsistencies,
**so that** the audit stories (12.1–12.3) work against a clean baseline and the codebase is maintainable.

## Acceptance Criteria
1. No `console.log` debug statements in admin panel source (legitimate logging excluded)
2. No `print()` debug statements in API source (legitimate logging via `logger` excluded)
3. No TODO/HACK/FIXME comments remaining (or each one is triaged and documented)
4. `ruff check` passes with zero errors on `apps/api/`
5. `npm run lint` passes with zero errors on `apps/admin-panel/`
6. All Python code formatted with `ruff format` (or black)
7. All TypeScript/TSX code formatted with `prettier`
8. `.gitignore` covers all dev artifacts (.env, __pycache__, node_modules, dist, .DS_Store)
9. No committed secrets or credentials in the repository
10. Clean git status after all changes committed

### Tooling
